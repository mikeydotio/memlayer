# Memlayer — Claude Code Memory Layer

## Context

Claude Code conversations are ephemeral — each session starts with no memory of prior work. The JSONL logs at `~/.claude/projects/**/*.jsonl` contain full conversation history but are unsearchable and deleted after 30 days. This project creates a persistent, semantically searchable memory layer that captures all conversations and makes them retrievable via MCP tools, giving Claude cross-session recall across all projects.

## Technology Choices

| Component | Language | Rationale |
|-----------|----------|-----------|
| **Daemon** | Rust | Single static binary, zero runtime deps, `notify` crate for file watching, Go unavailable |
| **Server API** | Python / FastAPI | `asyncpg` for raw SQL, OpenAI SDK, `uv` for fast deps, lowest LOC for the surface area |
| **MCP Server** | TypeScript | Official `@modelcontextprotocol/sdk` is TS-first, Node 22 available |
| **Database** | PostgreSQL 16 + pgvector | PRD requirement, HNSW for vectors, GIN for FTS |

## Phase 1: Infrastructure & Database

### 1.1 Docker Compose — `docker-compose.yml`

Two services: `db` (PostgreSQL + pgvector) and `server` (FastAPI). All ports bound to `${TAILSCALE_IP}` (default `0.0.0.0` for flexibility within Docker network, but server is only reachable over Tailnet in practice).

- `db`: `pgvector/pgvector:pg16`, volume `pgdata`, healthcheck via `pg_isready`
- `server`: built from `./server/Dockerfile`, depends on db healthy, port 8420
- Optional `ollama` service under `profiles: ["local-embeddings"]`

### 1.2 Database Migrations

**`001_extensions.sql`**
```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

**`002_tables.sql`**
```sql
CREATE TABLE IF NOT EXISTS claude_sessions (
    session_id VARCHAR(64) PRIMARY KEY,
    project_path VARCHAR(512),
    client_machine_id VARCHAR(128),
    slug VARCHAR(128),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_seen_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS memory_entries (
    id BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(64) NOT NULL REFERENCES claude_sessions(session_id),
    message_type VARCHAR(32) NOT NULL,      -- 'user', 'assistant'
    content_type VARCHAR(32) NOT NULL,      -- 'text', 'tool_use', 'tool_result'
    raw_content TEXT NOT NULL,
    payload_hash VARCHAR(64) NOT NULL UNIQUE, -- SHA-256 for idempotency
    source_uuid VARCHAR(64),
    parent_uuid VARCHAR(64),
    tool_name VARCHAR(128),                 -- populated for tool_use/tool_result blocks
    cwd VARCHAR(512),
    created_at TIMESTAMPTZ NOT NULL,        -- from JSONL timestamp
    ingested_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    fts tsvector GENERATED ALWAYS AS (to_tsvector('english', raw_content)) STORED,
    embedding vector(1536)                  -- NULL until async embedding runs
);
```

**`003_indexes.sql`**
- HNSW on `embedding` column: `USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 128)`
- GIN on `fts` column
- B-Tree on `session_id`, `created_at`, `payload_hash`
- B-Tree on `claude_sessions(project_path)`, `claude_sessions(slug)`

**`004_functions.sql`** — RRF hybrid search as stored function
```sql
CREATE OR REPLACE FUNCTION hybrid_search(
    query_text TEXT,
    query_embedding vector(1536),
    filter_session_id VARCHAR DEFAULT NULL,
    filter_project_path VARCHAR DEFAULT NULL,
    match_limit INT DEFAULT 20,
    fts_weight FLOAT DEFAULT 1.0,
    vector_weight FLOAT DEFAULT 1.0
) RETURNS TABLE (
    id BIGINT, session_id VARCHAR, message_type VARCHAR,
    raw_content TEXT, tool_name VARCHAR, created_at TIMESTAMPTZ,
    fts_rank INT, vector_rank INT, rrf_score FLOAT
) LANGUAGE sql STABLE AS $$
    WITH fts_results AS (
        SELECT me.id, ROW_NUMBER() OVER (ORDER BY ts_rank_cd(me.fts, websearch_to_tsquery('english', query_text)) DESC) AS rank
        FROM memory_entries me
        WHERE me.fts @@ websearch_to_tsquery('english', query_text)
          AND (filter_session_id IS NULL OR me.session_id = filter_session_id)
          AND (filter_project_path IS NULL OR me.session_id IN (
              SELECT cs.session_id FROM claude_sessions cs WHERE cs.project_path = filter_project_path))
        LIMIT 50
    ),
    vector_results AS (
        SELECT me.id, ROW_NUMBER() OVER (ORDER BY me.embedding <=> query_embedding) AS rank
        FROM memory_entries me
        WHERE me.embedding IS NOT NULL
          AND (filter_session_id IS NULL OR me.session_id = filter_session_id)
          AND (filter_project_path IS NULL OR me.session_id IN (
              SELECT cs.session_id FROM claude_sessions cs WHERE cs.project_path = filter_project_path))
        ORDER BY me.embedding <=> query_embedding
        LIMIT 50
    ),
    combined AS (
        SELECT COALESCE(f.id, v.id) AS id,
               COALESCE(f.rank, 51) AS fts_rank,
               COALESCE(v.rank, 51) AS vector_rank,
               (fts_weight / (60.0 + COALESCE(f.rank, 51))) +
               (vector_weight / (60.0 + COALESCE(v.rank, 51))) AS rrf_score
        FROM fts_results f FULL OUTER JOIN vector_results v ON f.id = v.id
    )
    SELECT me.id, me.session_id, me.message_type, me.raw_content,
           me.tool_name, me.created_at,
           c.fts_rank::INT, c.vector_rank::INT, c.rrf_score::FLOAT
    FROM combined c JOIN memory_entries me ON me.id = c.id
    ORDER BY c.rrf_score DESC LIMIT match_limit;
$$;
```

Also a `get_session_summary()` function returning chronological entries for a session.

**`db/init.sh`** — runs all `migrations/*.sql` in sort order via `psql`.

## Phase 2: Server API (Python/FastAPI)

### 2.1 Dependencies (`pyproject.toml`)
`fastapi`, `uvicorn[standard]`, `asyncpg`, `pgvector`, `openai`, `httpx`, `pydantic`, `pydantic-settings`

### 2.2 Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/api/ingest` | Bearer | Receive batch of parsed JSONL entries from daemon |
| `POST` | `/api/search` | Bearer | Hybrid search (called by MCP) |
| `GET` | `/api/sessions/{id}/summary` | Bearer | Chronological session entries |
| `GET` | `/health` | None | Health check |

### 2.3 Ingest Flow
1. Validate batch (max 200 entries per request)
2. For each unique `session_id`: `INSERT INTO claude_sessions ... ON CONFLICT (session_id) DO UPDATE SET last_seen_at = NOW()`
3. For each entry: `INSERT INTO memory_entries ... ON CONFLICT (payload_hash) DO NOTHING`
4. Return `{ accepted, duplicates, errors }`
5. Enqueue newly inserted IDs for async embedding

### 2.4 Embedding Worker
- In-process background task (not separate service)
- On startup: `SELECT id FROM memory_entries WHERE embedding IS NULL` to catch up
- Batches up to 100 texts per OpenAI API call
- Supports both OpenAI (`text-embedding-3-small`) and Ollama via config
- Falls back to FTS-only if no API key configured

### 2.5 Search Flow
1. Generate query embedding via provider
2. Call `hybrid_search()` stored function
3. Join with `claude_sessions` for project_path
4. Return results with RRF scores + timing metadata

### 2.6 Dockerfile
`python:3.12-slim` base, install deps via `uv pip install --system`, run with `uvicorn`.

## Phase 3: Client Daemon (Rust)

### 3.1 Dependencies (`Cargo.toml`)
`tokio` (full), `notify` (8.x), `serde`/`serde_json`, `sha2`, `reqwest` (json + rustls-tls), `rusqlite` (bundled), `dirs`, `tracing`, `glob`

### 3.2 JSONL Format (observed from actual logs)

Top-level keys: `parentUuid`, `isSidechain`, `userType`, `cwd`, `sessionId`, `version`, `gitBranch`, `type`, `message`/`data`, `uuid`, `timestamp`, `slug`, `isMeta`

Entry types:
- **`user`**: `message.content` is a string. Skip if `isMeta: true` or content starts with `<local-command-` / `<command-`. Otherwise extract the text as one entry.
- **`assistant`**: `message.content` is an array of blocks. Each block becomes a separate `memory_entries` row:
  - `type: "text"` → extract `.text` field, `message_type = "assistant"`, `content_type = "text"`
  - `type: "tool_use"` → `tool_name = .name`, content = JSON of `.input`, `content_type = "tool_use"`
  - `type: "tool_result"` → extract `.content` text, `content_type = "tool_result"`
  - `type: "thinking"` → **Skip** (contain signatures, not useful text)
- **`progress`**: Skip entirely (hook execution noise).
- **`file-history-snapshot`**: Skip entirely (file state tracking).

Project path decoded from directory name: `-home-mikey-memlayer` → `/home/mikey/memlayer`

### 3.3 Data Flow
```
~/.claude/projects/**/*.jsonl
    → FileWatcher (notify crate, recursive, debounce 500ms)
    → CursorManager (SQLite at ~/.local/share/memlayer/cursors.db, byte offsets)
    → Parser (read from offset, parse JSON lines, compute SHA-256, extract content)
    → Sender (POST batches of 50 to server /api/ingest, Bearer auth)
    → on failure → OfflineQueue (SQLite WAL at ~/.local/share/memlayer/queue.db)
    → drain loop every 30s with exponential backoff (1s→2s→4s→...→5min cap)
```

### 3.4 Key Behaviors
- **Startup catchup**: glob all `*.jsonl`, process from last cursor position
- **Log rotation**: JSONL files are append-only per session UUID. If file size < stored offset, reset to 0 (idempotency via payload_hash prevents duplicates)
- **Subagent files**: Skip files under `subagents/` directories (parent session captures the conversation)
- **Systemd**: user-level service at `~/.config/systemd/user/claude-mem-daemon.service` with `Restart=always`

## Phase 4: MCP Server (TypeScript)

### 4.1 Dependencies
`@modelcontextprotocol/sdk` (^1.27), `zod`

### 4.2 Tools

**`search_memory`**
- Params: `query` (string), `session_id?` (string), `project_path?` (string), `limit?` (number, default 10)
- Calls `POST /api/search` on server
- Returns formatted results with session ID, project, date, role, content snippet, RRF score

**`get_session_summary`**
- Params: `session_id` (string), `limit?` (number)
- Calls `GET /api/sessions/{id}/summary` on server
- Returns chronological conversation timeline

## Phase 5: Claude Skill — `skill/memory.md`

Teaches Claude when to use memory tools. Triggers on phrases like "do you remember", "how did we", "look up", "previous session". Instructs the pattern: search broadly → drill into session → synthesize. Installed to `~/.claude/skills/` or `~/.claude/commands/`.

## Key Design Decisions

1. **Configurable deployment**: Default to localhost for dev, document how to deploy to any host (brainframe, AWS, etc.). Server URL is a single env var.
2. **Bearer token auth**: Simple shared secret now, easy to swap for JWT/API keys when moving to cloud. No SSH tunnel dependency.
3. **Index everything**: User messages, assistant text, tool_use blocks (with tool_name), and tool_result blocks. Enables searching "what bash commands did we run" and "what did the file contain". Each meaningful content block becomes its own `memory_entries` row.
4. **Embedding as async background job**: Entries are FTS-searchable immediately on ingest. Embeddings generated asynchronously — never blocks ingestion.
5. **RRF in a stored function**: Single round-trip, no application-level merge. `STABLE` function, parameterized weights.
